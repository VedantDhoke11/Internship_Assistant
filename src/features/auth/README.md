# Authentication & Authorization Feature Module

This folder will house all authentication-related logic, routes, and components.

## Planned Stack
- Clerk (Third-party Authentication Provider)
- Next.js Middleware (Route Guarding)
- Custom onboarding hooks

## Directory Structure
- `/components`: Sign-in, sign-up forms, user buttons
- `/hooks`: useAuth, useSession wrapper hooks
- `/services`: Session handling, token management
- `/types`: Auth-specific TypeScript definitions
